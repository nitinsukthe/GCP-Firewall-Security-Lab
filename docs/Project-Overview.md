# Project Overview

## Project Title

GCP Firewall Security Lab: Securing Compute Engine with Least-Privilege Access

---

## Project Description

This project demonstrates the implementation of foundational cloud security controls within Google Cloud Platform (GCP). The lab focuses on securing a Compute Engine virtual machine using VPC Firewall Rules, network tags, and IP-based access restrictions.

The objective was to deploy a Linux-based virtual machine, restrict administrative access using the Principle of Least Privilege, validate secure connectivity through SSH, and perform proper cloud resource cleanup.

This project simulates real-world cloud security practices used by Cloud Security Engineers and Cloud Administrators to protect cloud workloads from unauthorized access.

---

## Objectives

- Create and configure a GCP project
- Enable Compute Engine API
- Create a custom VPC network
- Deploy a Compute Engine virtual machine
- Implement VPC Firewall Rules
- Restrict SSH access to a trusted public IP address
- Validate secure access using SSH
- Verify VM network configuration
- Perform secure resource cleanup
